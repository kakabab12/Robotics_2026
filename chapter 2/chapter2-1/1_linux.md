# 실습 1: 우분투 리눅스 환경 구성 및 기본 명령어 조사

## 1. 리눅스 기본 개념 조사

### 1) 셸 환경이 실행되는 터미널 실행 방법
- 우분투 데스크톱 환경에서 단축키 `Ctrl + Alt + T`를 누르면 기본 터미널(bash)이 실행됩니다.

### 2) 파일과 디렉토리를 관리하는데 필요한 명령어
- `ls`: 현재 디렉토리의 파일 및 폴더 목록 표시
- `cd [경로]`: 지정한 디렉토리로 이동
- `mkdir [이름]`: 새로운 디렉토리(폴더) 생성
- `rm [파일]`: 파일 삭제 (`-r` 옵션으로 디렉토리 삭제 가능)
- `cp [출발지] [목적지]`: 파일 또는 디렉토리 복사
- `mv [출발지] [목적지]`: 파일 위치 이동 또는 이름 변경

### 3) 파일의 내용을 확인하고 편집하는 방법
- **확인 명령:** `cat [파일명]` (전체 내용 출력), `less [파일명]` (화면 단위 확인)
- **편집 도구:** 터미널 내에서 `nano [파일명]` 또는 `vi [파일명]` 에디터를 사용하여 편집 가능

### 4) 파일의 권한에 대한 개념과 파일의 권한을 바꾸는 방법
- 리눅스 파일은 소유자(User), 그룹(Group), 기타 사용자(Others)별로 읽기(r), 쓰기(w), 실행(x) 권한을 가집니다.
- **권한 변경 명령어:** `chmod [권한설정] [파일명]` (예: `chmod +x 1_hello.py` 실행 권한 추가)

### 5) 실행파일, 셸 스크립트의 개념과 source 명령어 사용법
- **실행파일/셸 스크립트:** 리눅스에서 명령어들의 명령 집합을 텍스트 파일로 저장하여 실행할 수 있게 만든 파일입니다.
- **source 명령어:** 스크립트 파일을 현재 셸 환경에서 직접 실행하여, 스크립트 내부의 환경변수 설정 등이 현재 터미널 세션에 즉시 적용되도록 합니다. (예: `source ~/.bashrc`)

### 6) 사용자 권한, 슈퍼유저의 개념과 sudo 명령어
- **슈퍼유저 (Root):** 시스템의 모든 권한을 가진 최고 관리자 계정입니다.
- **sudo 명령어:** 일반 사용자가 시스템 설정이나 설치 등 관리자 권한이 필요한 작업을 수행할 때, 일시적으로 슈퍼유저 권한을 빌려 명령을 실행하게 해줍니다.

### 7) 우분투 리눅스의 패키지 관리 방법과 apt 명령어
- 우분투는 APT(Advanced Package Tool)를 통해 프로그램을 관리합니다.
- 패키지 목록 업데이트: `sudo apt update`
- 프로그램 설치: `sudo apt install [패키지명]`

### 8) 홈 디렉토리의 개념
- 로그인한 사용자 개인의 고유 작업 공간으로, 기호로 `~`로 표시됩니다. 일반 사용자는 이 홈 디렉토리(`~` 또는 `/home/유저명`) 내부에서만 자유롭게 파일을 생성하고 수정할 수 있습니다.

## 2. 파이썬 프로그램 실행 및 결과 확인

### 1) 예외 없는 동작을 위한 실행 방법
- 본 프로그램은 리눅스의 루트 디렉토리 하위인 `/test`에 접근하므로 일반 권한으로는 `PermissionError`가 발생합니다.
- 이를 예외 없이 정상 동작시키기 위해 **슈퍼유저 권한(`sudo`)**을 사용하여 실행합니다.
```bash
sudo python3 1_hello.py
