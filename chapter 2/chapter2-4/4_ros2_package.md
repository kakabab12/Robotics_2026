ROS2 패키지 및 파이썬 빌드 시스템 학습 보고서
1. colcon 빌드 도구

    개념: colcon은 ROS2에서 표준으로 사용하는 빌드 도구입니다. 워크스페이스 내의 여러 패키지를 한 번에 빌드하고, 각 패키지의 의존성을 관리하며 설치 과정을 자동화합니다.

    역할: 소스 코드를 컴파일하거나 파이썬 패키지를 설치 가능한 형태로 정리하여 install/ 디렉토리에 배치합니다.

2. rclpy (ROS2 Python Client Library)

    개념: ROS2의 파이썬용 클라이언트 라이브러리입니다.

    용도: 파이썬 환경에서 ROS2의 노드 생성, 토픽 발행(Publish) 및 구독(Subscribe), 서비스(Service), 액션(Action) 등을 구현할 수 있게 해주는 핵심 라이브러리입니다. ROS2 제어 프로그램을 파이썬으로 작성할 때 필수적으로 의존성을 추가해야 합니다.

3. ROS2 패키지의 핵심 파일

    package.xml:

        역할: 패키지의 메타데이터(설명서)입니다. 패키지 이름, 버전, 작성자 정보, 라이선스, 그리고 이 패키지가 실행되기 위해 필요한 의존성 패키지들(rclpy 등)을 선언합니다.

    setup.py:

        역할: 파이썬 패키지의 설치 설정 파일입니다. 빌드 시스템이 이 패키지를 어떻게 설치할지 결정하며, 특히 노드를 실행할 수 있도록 entry_points를 지정하여 추후 ros2 run 명령으로 실행 가능하게 만듭니다.
