# 6_callback.md

## 1. 타이머와 콜백 개념 조사
- **타이머(Timer)**: ROS2 시스템에서 특정 시간 간격(주기)마다 정의된 함수(콜백)를 자동으로 실행하도록 만드는 객체입니다. `self.create_timer(주기, 콜백함수)` 형태로 생성하며, 센서 데이터 수집, 로봇 상태 업데이트 등 주기적인 제어가 필요한 작업에 필수적으로 사용됩니다.
- **콜백(Callback)**: 이벤트(타이머 만료, 토픽 수신 등)가 발생했을 때 호출되는 함수를 말합니다. 로봇 시스템은 메인 루프에서 모든 것을 처리하는 것이 아니라, 이러한 콜백 함수들을 비동기적으로 호출하여 효율적으로 다중 작업을 수행합니다.

## 2. 노드 구현 (`timer_test.py`)
- **클래스 정의**: `TimerNode` 클래스를 생성하여 `counter`라는 변수를 0으로 초기화했습니다.
- **타이머 생성**: 2초 주기(`timer2`)와 3초 주기(`timer3`) 타이머를 생성하고, 각각 서로 다른 콜백 함수(`timer_callback_2s`, `timer_callback_3s`)를 연결했습니다.
- **로직 구현**:
    - 2초 콜백: `counter`를 1 증가시키고 로그 출력.
    - 3초 콜백: `counter`를 1 감소시키고 로그 출력.

## 3. 실행 결과 확인
- `ros2 run my_robot_controller timer_node` 명령 실행 시, 터미널 로그를 통해 2초와 3초 주기로 로그 메시지가 출력되는 것을 확인하였습니다.
- `counter` 값이 두 콜백의 실행에 따라 정상적으로 증감하는 것을 통해 두 개의 타이머가 독립적으로, 그러나 공유 자원(`self.counter`)을 안전하게 제어하고 있음을 확인했습니다.

## 4. 실습 소감
- 로봇의 동작 주기를 제어하는 타이머와 이벤트 처리 방식인 콜백 개념을 실습하며, ROS2가 어떻게 효율적인 비동기 처리를 수행하는지 이해하게 되었습니다.
- 특히 실습 과정에서 발생한 패키지 경로 문제(워크스페이스와 과제 저장소 분리)와 빌드 캐시 문제들을 해결하며, ROS2 개발 환경에서의 필수적인 관리 절차를 확실히 숙지했습니다. 로봇 소프트웨어의 본질인 '멈추지 않는 지속적인 데이터 처리'를 경험한 의미 있는 단계였습니다.
