# [과제 보고서] ROS2 비동기 서비스 호출 및 데드락 방지 구현

## 1. 과제 목적 및 배경
* **목적**: ROS2 환경에서 서비스 서버와 클라이언트 역할을 동시에 수행하는 노드를 설계하고, 특정 서비스 호출(`~/quit`) 시 다른 서비스(`/kill`)를 연쇄적으로 호출하여 시스템을 안전하게 종료(Clean Shutdown)하는 메커니즘을 구현함.
* **문제 핵심**: 서비스 콜백 함수 내부에서 또 다른 서비스를 호출할 때 발생하는 **데드락(Deadlock)** 문제를 이해하고, 이를 **비동기 호출 방식(`call_async`)**으로 해결하는 능력을 배양함.

---

## 2. 핵심 기술 개념 요약

### 2.1. 동기식 호출(Sync Call)의 한계와 데드락 발생 원인
1. **동기식 호출(`call`)의 특징**: 서비스 요청을 보낸 후 서버로부터 응답(Response)이 올 때까지 현재 스레드의 실행을 블로킹(Blocking)하고 대기함.
2. **데드락(Deadlock) 메커니즘**:
   * ROS2의 기본 실행기인 **단일 스레드 실행기(Single-Threaded Executor)**는 한 번에 하나의 콜백만 처리할 수 있음.
   * `/quit` 서비스 요청이 들어오면 실행기는 해당 콜백 스레드를 점유함.
   * 콜백 내부에서 `/kill` 서비스를 동기식(`call`)으로 호출하면, 실행기는 `/kill` 응답이 올 때까지 대기(블로킹) 상태에 빠짐.
   * 하지만 `/kill` 서비스의 응답을 받아 처리해야 할 실행기(Executor)가 이미 `/quit` 콜백에 의해 붙잡혀 있으므로, 응답 처리가 불가능해짐.
   * 결과적으로 두 프로세스가 서로를 기다리는 **무한 대기(데드락)** 상태에 빠지며 노드가 멈추게 됨.

### 2.2. 비동기식 호출(Async Call)을 통한 해결 방안
* **비동기식 호출(`call_async`)의 특징**: 서비스 요청을 보낸 즉시 제어권을 실행기(Executor)에 반환하며, 미래에 올 응답 객체인 `Future`를 즉시 리턴함. 스레드를 블로킹하지 않으므로 노드의 메인 주행 루프(토픽 퍼블리시/서브스크라이브)가 중단 없이 유지됨.
* **완료 콜백(`add_done_callback`) 활용**:
  * `call_async()` 호출 후 응답 처리를 강제로 기다리지 않고, 응답이 도착했을 때 실행할 별도의 콜백 함수(`kill_done_callback`)를 지정함.
  * 이를 통해 실행기는 자유롭게 다른 이벤트(응답 수신 등)를 처리할 수 있으며, 데드락 구조를 원천적으로 차단함.

## 3. 동기 vs 비동기 메커니즘 비교

| 비교 항목 | 동기식 호출 (`call`) | 비동기식 호출 (`call_async`) |
| :--- | :--- | :--- |
| **스레드 상태** | 응답이 올 때까지 블로킹 (Blocking) | 즉시 제어권 반환 (Non-Blocking) |
| **데드락 위험성** | 단일 스레드 환경에서 발생 확률 매우 높음 | 구조적으로 데드락 방지 가능 |
| **로봇 주행 영향** | 서비스 대기 중 토픽 발행 등이 중단됨 | 서비스 호출 중에도 정상 주행 가능 |
| **응답 처리 방식** | 리턴 값을 직접 받음 | `Future` 객체 및 `Done Callback` 활용 |

## 4. 시스템 클린 셧다운(Clean Shutdown) 구현 결과
* **안전한 자원 해제**: 비동기 콜백을 통해 `turtlesim` 내부의 거북이 개체(`/turtle1`)를 완전히 제거한 것이 확인된 시점에 `rclpy.shutdown()`을 호출함.
* **예외 처리 검증**: 종료 과정에서 프로세스가 강제 크래시(Crash)되거나 `Unhandled Exception`을 발생시키지 않고, 터미널 프롬프트로 깔끔하게 복귀하는 **클린 셧다운(Clean Shutdown)** 프로세스를 완수함.

> **결론 및 시사점**
> ROS2 기반의 로봇 시스템은 수많은 이벤트와 서비스가 동시다발적으로 상호작용하므로, 자칫 유기적인 서비스 호출이 시스템 전체를 마비시키는 데드락을 유발할 수 있다. 본 과제를 통해 단일 스레드 실행기의 동작 원리를 이해하고, `call_async`와 완료 콜백 메커니즘을 적용하여 실시간성(Real-time)을 보장하는 비동기 제어 노드를 성공적으로 구현하였다.
