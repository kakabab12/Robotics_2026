
# ROS2 파이썬 노드 구현 실습 보고서

## 1. 노드 구현 핵심 이론
- **Node 클래스 상속**: 모든 ROS2 노드는 `rclpy.node.Node`를 상속받아 구현합니다. 이를 통해 노드 이름 설정, 로거 사용, 통신 기능(토픽/서비스)을 활용할 수 있습니다.
- **주요 함수**:
    - `rclpy.init()`: ROS2 통신을 시작하기 위해 클라이언트 라이브러리를 초기화합니다.
    - `rclpy.spin()`: 노드를 실행 상태로 유지하며, 콜백 함수를 처리합니다.
    - `rclpy.shutdown()`: 노드 종료 시 자원을 깔끔하게 해제합니다.
- **로깅(Logging)**: `print()` 대신 `self.get_logger().info()`를 사용합니다. 이를 통해 ROS2의 시스템 로그 관리 체계와 연동되어 실행 환경에 기록을 남길 수 있습니다.

## 2. 실습 코드 (logging.py)
- `my_robot_controller` 패키지 내 `logging.py` 파일을 생성하여 구현했습니다.
- `logging_node`라는 이름의 노드를 생성하고, 실행 시 로그를 출력하며 `spin`을 통해 동작 상태를 유지하도록 했습니다.

## 3. 빌드 및 실행
- `--symlink-install` 옵션을 사용하여 빌드했습니다. 이 옵션은 소스 코드가 수정될 때마다 전체를 다시 빌드할 필요 없이, 소스 코드의 링크를 직접 참조하게 하여 개발 효율을 높여줍니다.
- 실행 결과, 터미널에서 성공적으로 로그 메시지가 출력되는 것을 확인했습니다.

# 5_ros2_python_node.md

## 1. 노드 클래스 및 rclpy 라이브러리 조사
- **Node 클래스 상속**: `rclpy.node.Node`를 상속받아 노드를 정의하며, 이를 통해 ROS2의 핵심 기능을 사용합니다.
- **주요 함수**:
    - `rclpy.init()`: ROS2 시스템 초기화.
    - `rclpy.spin()`: 노드를 실행하여 콜백 함수를 대기(루프).
    - `rclpy.shutdown()`: ROS2 시스템 종료 및 자원 해제.
- **로그 기록**: `self.get_logger().info()`를 사용하여 표준 출력이 아닌 ROS2 로깅 시스템을 통해 메시지를 기록했습니다.

## 2. 노드 구현 (`logging_node.py`)
- `my_robot_controller` 패키지 내에 작성하였으며, 노드 이름은 `logging_node`로 설정하였습니다. 
- 실행 시 "Logging node has been started!"라는 로그가 INFO 레벨로 출력되도록 구현하였습니다.

## 3. 빌드 및 실행 결과
- `colcon build --symlink-install` 명령어를 사용하여 빌드하였고, `--symlink-install` 옵션을 통해 소스 변경 시 재빌드 없이 바로 반영되도록 설정하였습니다.
- `ros2 run my_robot_controller logging_node` 실행 시 성공적으로 로그가 출력됨을 확인하였습니다.

## 4. 실습 소감
- 파이썬으로 직접 노드를 생성하고 빌드하여 실행해보며 ROS2 시스템의 동작 원리(노드 간 통신 기반의 아키텍처)를 이해할 수 있었습니다. 특히 파일 이름 충돌(logging 모듈)과 빌드 캐시 문제 등을 해결하며, ROS2 환경에서의 개발 주의사항을 확실히 배웠습니다.
