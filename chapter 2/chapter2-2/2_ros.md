# ROS2 실습 보고서 (과제 2/2)

## 1. ROS2 이론 조사

### 1) 로봇 운영체제(ROS)의 개념
ROS(Robot Operating System)는 로봇 응용 프로그램을 개발하기 위한 오픈소스 미들웨어 프레임워크입니다. 하드웨어 추상화, 저수준 장치 제어, 프로세스 간 메시지 전달(Pub/Sub), 패키지 관리 등 로봇 개발에 필수적인 기능을 제공합니다.

### 2) 운영체제 사용 로봇 vs 미사용 로봇 차이
- **운영체제 사용(OS-based):** 복잡한 로봇 소프트웨어를 모듈화하여 관리할 수 있고, 멀티태스킹이 가능합니다. AI, 자율주행 등 고수준 알고리즘 구현이 용이합니다.
- **운영체제 미사용(Embedded/Bare-metal):** 하드웨어 제어 속도가 빠르고 실시간성(Real-time)이 보장되지만, 기능 확장이 어렵고 복잡한 연산을 구현하기 어렵습니다.

### 3) 패키지와 노드의 관계
- **패키지(Package):** ROS2 소프트웨어의 구성 단위로, 특정 기능을 수행하기 위한 코드, 설정 파일, 라이브러리 등이 모여 있는 폴더입니다.
- **노드(Node):** 패키지 내에서 하나의 실행 단위(프로세스)입니다. 로봇의 각 센서나 모터 제어 등 특정 기능을 수행하며, 서로 데이터를 주고받으며 로봇 시스템을 구성합니다.

### 4) `ros2 run` 명령어 사용법
- 패키지 내의 실행 가능한 노드를 실행하는 명령어입니다.
- 문법: `ros2 run [패키지_이름] [실행파일(노드)_이름]`
- 예시: `ros2 run demo_nodes_cpp talker`
