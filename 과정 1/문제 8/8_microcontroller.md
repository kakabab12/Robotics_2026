## 마이크로컨트롤러 및 제어 시스템 이해


1. 마이크로컨트롤러(MCU)의 역할
마이크로컨트롤러는 CPU, 메모리, 입력/출력(I/O) 주변장치를 하나의 칩에 통합한 소형 컴퓨터입니다. 로봇에서는 센서 데이터를 수집하고, 상황을 판단하며, 모터와 같은 액추에이터를 정밀하게 제어하는 '중추 신경' 역할을 수행합니다.

2. 기본 제어 방식 및 인터페이스
PWM (Pulse Width Modulation): 디지털 신호를 고속으로 스위칭하여 평균 전압을 조절하는 방식입니다. 모터의 속도 제어나 LED 밝기 조절에 필수적입니다.

디지털 핀: 0(Low) 또는 1(High)의 전압 상태만 가집니다. 버튼 입력이나 모터 드라이버의 방향 제어에 사용됩니다.

아날로그 핀 (ADC): 연속적인 전압 값을 디지털 숫자로 변환합니다. 센서의 값을 읽어올 때 사용됩니다.

3. 마이크로컨트롤러의 통신 방식
로봇 시스템은 여러 센서 및 모듈과 데이터를 주고받기 위해 다양한 통신 프로토콜을 사용합니다.

UART (Universal Asynchronous Receiver/Transmitter): 1:1 통신. 가장 단순하며 PC와 아두이노 간의 시리얼 통신에 주로 사용됩니다.

I2C (Inter-Integrated Circuit): 두 선(SDA, SCL)으로 여러 장치를 연결합니다. 센서 모듈 연결에 주로 사용됩니다.

SPI (Serial Peripheral Interface): 고속 통신이 필요할 때 사용됩니다. 데이터 전송 속도가 빠르지만 핀을 많이 점유합니다.

4. 실시간 제어와 타이머
마이크로컨트롤러는 정해진 주기(Sample time)마다 센서를 읽고 제어 값을 계산해야 합니다.

타이머/카운터: 내부 클럭을 사용하여 일정한 시간 간격을 측정하거나 주기적인 인터럽트를 발생시킵니다.

사용 절차: 타이머 설정 -> 클럭 소스 선택 -> 비교 값 설정 -> 인터럽트 활성화 -> ISR(Interrupt Service Routine) 수행.

5. 아두이노(Arduino)
개념: 오픈 소스 하드웨어 플랫폼으로, 복잡한 MCU 설정을 라이브러리와 추상화된 함수로 쉽게 사용할 수 있게 합니다.

인터페이스: setup() (초기화)과 loop() (반복 수행)라는 단순한 구조를 통해 로봇 제어 코드를 직관적으로 작성할 수 있습니다.
